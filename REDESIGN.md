## 2. フォロワーが増えない構造的な理由 ★最重要

### 2-1. 全投稿が「宣伝」になっている

`generate_tree.py` の SYSTEM_PROMPT に以下がハードコードされている:

```
- 最後の投稿に必ずブログURLを「👇 続きはこちら\n<URL>」の形で入れる。
- ハッシュタグは最後の投稿に2〜3個まで（#ライフハック #AI活用 #時短 など）。
```

**例外なく100%の投稿が外部リンクで終わる。**
決めてあるバランス（有益4 : 経験談5 : 宣伝1）に対して、現状は宣伝10。

why: Threads は外部リンク付き投稿のリーチを構造的に絞る。さらに読者視点では、
「毎回ブログに来いと言ってくるアカウント」をフォローする理由がない。
フォローは「この人の投稿がタイムラインに流れてくると嬉しい」と思われて初めて発生する。
今の設計は、その状態を1回も作らないまま送客だけを試み続けている。

### 2-2. ハッシュタグの運用がプラットフォーム違い

Threads はトピックタグを**1投稿に1つ**しか実質機能させない。
2〜3個の羅列は X の作法で、Threads では効果がないどころか業者っぽく見える。

### 2-3. フォローする理由（フック）が設計に無い

今あるのは「記事 → ツリー化」だけ。
毎回テーマもトーンもバラバラで、「この人は何の人か」が3投稿見ても分からない。
連載・定型フォーマット・キャラクターのいずれも無い。

### 2-4. 発信オンリーで、交流の設計がゼロ

Threads でフォロワーが増える最大の経路は、**自分より大きいアカウントへの返信**。
現在のパイプラインには返信を扱うコンポーネントが一切ない。
投稿だけを自動化して、伸びる要因の方を自動化していない。

### 2-5. 指標が「フォロワー」を向いていない

`posted_log.json` に記録されるのは投稿ID だけ。
仮に views/likes を取り始めても、それは「フォロワーが増えたか」とは別の指標。
バズったがフォロワーは1人も増えない投稿は普通にある。

## 3. 目指す形

### 3-1. パイプライン

```
[21:00] 生成      … 記事在庫から1本 + 投稿タイプを配分ルールで決定
           ↓ 機械フィルタ（内部ラベル/定型/重複を自動リジェクト）
[毎晩30秒] 承認   … Notionで「面白いか」だけ○×
           ↓
[08:00]   投稿   … 直前に重複最終チェック（防衛線）
           ↓
[翌々日]   回収   … insights + フォロワー数の日次スナップショット
           ↓
[月曜21:00] 週次  … 投稿タイプ別・切り口別にフォロワー貢献を答え合わせ
```

### 3-2. 投稿タイプの配分（新設）

1週間7本を、以下の比率で自動配分する:

| タイプ | 本数/週 | 内容 | 外部リンク |
|---|---|---|---|
| 経験談 | 3〜4 | 自分が実際にやって損した/得した話。記事が元ネタでも**リンクを貼らない** | なし |
| 有益 | 2〜3 | その場で完結する知識。読んで得して終わり | なし |
| 宣伝 | 1 | ブログ送客。「続きはこちら」を使うのはこの1本だけ | あり |

why: 「リンクなしで価値を渡す投稿」が週6本あって初めて、
週1本の送客投稿が信頼の上に乗る。今は信頼の蓄積フェーズが丸ごと抜けている。

### 3-3. 決定事項
- 投稿頻度: **1日1本**（朝スロットのみ）
- 承認: **機械フィルタ + 人は面白さだけ○×**（毎晩30秒）
- ハッシュタグ: **0〜1個**（Threads の作法に合わせる）
- 外部リンク: **週1本の宣伝投稿のみ**

## 4. 指標の再設計 ★重点

「チェックして修正する」を回すための計測。**フォロワーを頂点に置く。**

### 主指標（これが伸びなければ全部失敗）
- **週次フォロワー増加数** … 毎日フォロワー数をスナップショットして差分を取る

### 副指標（主指標の原因を説明するために取る）
- 投稿別 views / likes / replies（Threads Insights API）
- **プロフィール表示数** … Threads Insights の `follows` / プロフィールクリック系メトリクス。
  「投稿を見た人が、この人を知りたくなったか」= フォローの直前段階
- 投稿タイプ別（経験談/有益/宣伝）の平均パフォーマンス
- 切り口別（逆張り/失敗談/数字ショック等、既存の10種）の平均パフォーマンス

### 答え合わせの単位
週次で「フォロワーが増えた日」と「その日に出した投稿のタイプ・切り口」を突き合わせる。
2〜3週で、伸びる切り口とそうでない切り口の差が見えるはず。
**そこで初めて ANGLES のリストを削り込む**（今は10種を漫然と回しているだけ）。

## 6. 進め方の原則

- **P2（計測）が動くまで、投稿内容の細かい作り込みに時間を使わない**
  数字が取れないまま質をいじると、改善したつもりで3ヶ月溶ける
- ただし **P1 は計測を待たずにやる**。
  「全投稿が宣伝」は計測するまでもなく構造的に間違っているため
- 1セッション1.5h以内。P0 → P1 → P2 の順に、1回のセッションで1ブロック
